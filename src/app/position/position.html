🗂 Entity Fields — Position
🔑 PositionID — Primary Key 🏷 Title — Job Title 🔗 DeptID — FK → Department 🔗 UserID — FK → User (nullable = vacant position)
💡 Design Note: Because UserID lives on the Position entity (not User), one user can hold multiple positions simultaneously, and a position can be vacant (UserID = NULL).
